展示模式控制

統一家庭代碼，再開 App Demo / Dashboard。

直接用 file:// 開啟控制台時，Chrome 不允許它直接寫入擴充功能 Popup 的 chrome.storage。 這版仍會把 familyID 帶到 App Demo / Dashboard；若要 Popup 也同碼，請先從 welcome.html 或 Popup 建立/綁定家庭代碼，再按「重新讀取家庭代碼」。

備援模式會設定 localStorage 與網址參數 ?forceFallback=1。

決賽展示流程

依照這個順序展示，讓評審快速理解痛點、技術架構與實際防護流程。

1　痛點說明 50+ 使用者遇到可疑訊息時，常因限時、付款、一次性碼與高報酬話術而被催促操作。
2　系統狀態確認 透過展示控制台確認後端 API、AI 分析、ScamDNA / RAG、雙語語音與家庭戰情室皆已就緒。
3　App Demo 掃描 選擇高風險案例，一鍵帶入 App Demo，立即掃描後顯示風險分數、判斷理由與雙語提醒。
4　家庭聯防展示 開啟 Dashboard，展示同一家庭代碼下的掃描紀錄、風險狀態與家庭守護流程；正式模式可推播 LINE 通知家人。
5　容錯與備援 切換本機備援模式，說明即使雲端 API 或網路不穩，系統仍可維持基本防護。
